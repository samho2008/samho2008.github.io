# 检查是否安装 Docker Compose

查看安装 Docker 安装的时候是否安装了 Docker Compose 组件。

docker compose version

[root@localhost ~]# docker compose version
Docker Compose version v2.10.2

如果显示命令不存在，需要先安装 docker-compose，参考文档：https://bbs.huaweicloud.com/blogs/286823 (opens new window)。

# 创建 docker-compose.yml

创建一个 docker-compose.yml 文件，输入【i】进入编辑模式。

vi docker-compose.yml

把以下内容复制进去。

其中，services.server.image 为使用的镜像，你可能需要修改版本号，下面以【1.17.2】为例。

注意：所有数据均保存到宿主机上的【/usr/local/gitea】目录下。（如果该目录不存在会自动创建）

version: "3"

networks:
  gitea:
    external: false

services:
  server:
    image: gitea/gitea:1.17.2
    container_name: gitea
    environment:
      - USER_UID=1000
      - USER_GID=1000
    restart: always
    networks:
      - gitea
    volumes:
      - /usr/local/gitea:/data
      - /etc/timezone:/etc/timezone:ro
      - /etc/localtime:/etc/localtime:ro
      - /home/git/.ssh/:/data/git/.ssh
    ports:
      - "3000:3000"
      - "127.0.0.1:222:22"

# SSH 端口转发

如果使用的 Docker 部署，由于容器和主机环境隔离，不能直接使用 22 端口，需要把主机的 22 端口转发到特定的端口。

参考文档：

Installation with Docker (opens new window)

Docker 安装 Gitea/Gogs 与主机共享 22 端口 (opens new window)

以下命令都在主机上执行：

# 创建 Git 用户
adduser git

# 以 git 用户身份创建 SSH 密钥
sudo -u git ssh-keygen -t rsa -b 4096 -C "Gitea Host Key"

# =================== 注意，这里要连续按三下回车确认生成的密钥信息，不要把下面的内容和上面命令一起复制

# 把公钥设置为授权的密钥
echo "no-port-forwarding,no-X11-forwarding,no-agent-forwarding,no-pty $(cat /home/git/.ssh/id_rsa.pub)" >> /home/git/.ssh/authorized_keys

# authorized_keys 文件需要 600 权限
chmod 600 /home/git/.ssh/authorized_keys

# 创建文件，用来端口转发用
cat <<"EOF" | sudo tee /usr/local/bin/gitea
#!/bin/sh
ssh -p 222 -o StrictHostKeyChecking=no git@127.0.0.1 "SSH_ORIGINAL_COMMAND=\"$SSH_ORIGINAL_COMMAND\" $0 $@"
EOF

# 添加执行权限
sudo chmod +x /usr/local/bin/gitea

查看 git 用户的 UID 和 GID

id git


[root@localhost ~]# id git
uid=1000(git) gid=1000(git) 组=1000(git)

修改 docker-compose.yml，把 222 端口转发到容器内部的 22 端口，并且这个端口是用来转发用的，所以只需要本机使用即可，不用暴露到公网。

vi docker-compose.yml

在 volumes 中添加：

      - /home/git/.ssh/:/data/git/.ssh

在 ports 中添加：

      - "127.0.0.1:222:22"

如图所示，注意这里的 UID 和 GID 要和上面 git 用户的 USER_UID 和 USER_GID 的值对应！

# 配置 Gitea

访问 http://ip:3000，把【ip】换成服务器的地址。

数据库使用默认的【SQLite3】即可，

【站点名称】可以改成自己的公司或者团队名字，

【服务器域名】和【基础URL】中的【localhost】要改成服务器 IP 地址，或者用域名。

注意：这里的【SSH 服务端口】（后面会做 SSH 端口转发，这里可以用 22，虽然是 222:22，但是不会和主机的冲突）和【HTTP 服务端口】要和主机上的端口对应，而不是用容器内部的端口。

比如 HTTP 服务端口默认是【3000:3000】，如果修改为【5000:3000】，那么这里的 HTTP 服务端口也要修改为【5000】！

# Nginx 反向代理

由于我们是使用 Docker 部署的，需要使用 Nginx 进行域名绑定和端口转发。

vi /usr/local/nginx/conf/nginx.conf

把所有通过 IP 访问 80 端口的请求都转发到域名，这样通过 IP 访问就会直接跳转到域名。

下面以【git.team.com】域名为例子。

  server {
    listen 80;
    server_name localhost;
    rewrite ^/(.*)$ http://git.team.com/$1 permanent;
  }

所有访问【git.team.com】的请求都转发到 3000 端口上。（Gitea 的 Web 服务使用的是 3000 端口）

  server {
    listen 80;
    server_name git.team.com;
    location / {
      proxy_pass http://localhost:3000;
    }
  }

这个时候，你会发现一个问题，直接通过【IP:3000】的方式，还是可以访问到 Gitea，这是因为我们使用 Docker 搭建的容器，如果直接通过【3000:3000】这种方式映射端口，宿主机上的端口会自动放行 iptables 防火墙上的端口（CentOS 7 引入的 firewalld 防火墙，CentOS 6 引入的 iptables，CentOS 7 的 firewalld 和 iptables 两个防火墙可以兼容，而 Docker 使用的是 iptables 防火墙），那么，如果我们想 3000 端口只能在本机访问，外网不能访问要怎么操作呢？

修改 docker-compose.yml。

把 ports 中原来【3000:3000】改为【127.0.0.1:3000:3000】，表示宿主机上的端口只能在【127.0.0.1】访问，也就是本机。而默认是 0.0.0.0 所有主机都可以连接。

修改完成后，把原来的容器删掉，利用现在的文件重新搭建一个新的，别担心，你的数据都挂载到了宿主机上，不会丢失。

# 删除现在的 Gitea 容器
docker rm -f gitea
# 使用现在的脚本重新搭建一个新的容器
docker compose up -d
